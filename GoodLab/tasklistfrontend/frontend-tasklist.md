# GoodLab Frontend Implementation Tasklist

> 프론트엔드 구현을 위한 태스크 리스트 (백엔드 제외)
> 우선순위: 높음(🔴), 중간(🟡), 낮음(🟢)

---

## Phase 1: 프로젝트 기초 설정

### Task 1: 프로젝트 초기 셋업 🔴
**소요 시간**: 2-3 시간

- [ ] Next.js 14 프로젝트 생성 (App Router)
- [ ] TypeScript 5+ 설정
- [ ] Tailwind CSS 3+ 설치 및 설정
- [ ] shadcn/ui 초기화 및 기본 컴포넌트 설치
  - Button, Card, Input, Label, Select, Dialog 등
- [ ] 프로젝트 폴더 구조 설정
  ```
  /app
    /auth
    /dashboard
    /admin
    /professor
    /team
  /components
    /ui (shadcn)
    /layout
    /features
  /lib
  /hooks
  /store
  /types
  ```
- [ ] ESLint, Prettier 설정

**완료 기준**: `npm run dev` 실행 시 정상 작동, shadcn/ui 컴포넌트 import 가능

## Phase 2: 인증 및 레이아웃

### Task 2: 인증 UI 구현 🔴
**소요 시간**: 4-5 시간

- [ ] 로그인 페이지 (`/auth/login`)
  - 이메일/비밀번호 입력 폼
  - Google OAuth 버튼 (선택)
  - "비밀번호 찾기" 링크
- [ ] 회원가입 페이지 (`/auth/signup`)
  - 이메일/비밀번호/이름 입력
  - 프로필 이미지 업로드 (선택)
- [ ] 비밀번호 재설정 페이지
- [ ] React Hook Form + Zod 스키마 적용
- [ ] 폼 검증 및 에러 메시지 표시

**완료 기준**: 모든 인증 페이지 UI 완성, 폼 검증 작동

### Task 3: 기본 레이아웃 컴포넌트 🔴
**소요 시간**: 3-4 시간

- [ ] 헤더 컴포넌트
  - 로고
  - 유저 프로필 드롭다운 (이름, 아바타, 로그아웃)
  - 알림 아이콘
- [ ] 사이드바 컴포넌트 (Slack 스타일)
  - 내 방 목록

  - 문서 섹션
  - 설정 링크
  - 모바일: 햄버거 메뉴로 토글
- [ ] 메인 레이아웃 컴포넌트
  - 사이드바 + 헤더 + 콘텐츠 영역
- [ ] 반응형 레이아웃 (모바일/태블릿/데스크탑)

**완료 기준**: 레이아웃이 모든 화면 크기에서 정상 작동, 사이드바 토글 기능 작동

### Task 5: 방(Room) 관리 UI 🔴
**소요 시간**: 5-6 시간

  - React Hook Form + Zod
- [ ] 방 수정/삭제 기능
- [ ] 방 상세 페이지 (`/professor/rooms/[id]`)
  - 방 정보 표시
  - 팀 목록 표시

**완료 기준**: 방 CRUD UI 완성, 폼 검증 작동

- [ ] 문서 목록 페이지 (`/documents`)

  - 현재 편집 중인 유저 표시

## Phase 6: 최적화 및 마무리

### Task 13: 권한 기반 라우팅 및 가드 🔴
**소요 시간**: 3-4 시간

- [ ] 권한 체크 훅 (`useAuth`, `useRole`)
- [ ] 라우트 가드 컴포넌트
  - `ProtectedRoute`: 로그인 필수
  - `AdminRoute`: 교수 권한 필요
  - `SuperAdminRoute`: 슈퍼관리자 권한 필요
- [ ] 권한 없음 페이지 (`/403`)
- [ ] 각 페이지에 라우트 가드 적용
- [ ] 권한별 사이드바 메뉴 필터링
  - 슈퍼관리자: 모든 메뉴
  - 교수: 방/팀 관리 메뉴
  - 팀장: 팀 설정 메뉴
  - 일반 유저: 대시보드, 문서

**완료 기준**: 권한별로 페이지 접근 제어 작동, 권한 없을 시 403 페이지 표시

---

### Task 14: 전역 상태 관리 (Zustand) 🔴
**소요 시간**: 3-4 시간

- [ ] Zustand 스토어 설정
  - `useAuthStore`: 유저 정보, 로그인 상태
  - `useRoomStore`: 현재 선택된 방
  - `useTeamStore`: 현재 선택된 팀
  - `useAnalysisStore`: 분석 상태
  - `useDocumentStore`: 문서 편집 상태
- [ ] 스토어 persist 설정 (localStorage)
- [ ] 각 컴포넌트에 스토어 연결
- [ ] 스토어 액션 정의
  - 로그인/로그아웃
  - 방/팀 선택
  - 분석 시작/완료

**완료 기준**: Zustand 스토어로 전역 상태 관리, persist 작동

---

### Task 15: 반응형 디자인 및 최적화 🟢
**소요 시간**: 4-5 시간

- [ ] 모바일 레이아웃 최적화 (< 640px)
  - 사이드바 햄버거 메뉴로 변경
  - 테이블 → 카드 레이아웃
  - 터치 친화적 버튼 크기
- [ ] 태블릿 레이아웃 (640px - 1024px)
  - 축소된 사이드바
  - 그리드 레이아웃 조정
- [ ] 데스크탑 레이아웃 (> 1024px)
  - 전체 레이아웃 활용
- [ ] 이미지 최적화 (Next.js Image)
- [ ] 코드 스플리팅 및 Lazy Loading
- [ ] 라이트하우스 성능 테스트
  - Performance > 90
  - Accessibility > 95

**완료 기준**: 모든 화면 크기에서 정상 작동, 라이트하우스 점수 목표 달성

## 참고 사항

### 기술 스택 (확인)
- Next.js 14+ (App Router)
- TypeScript 5+
- Tailwind CSS 3+
- shadcn/ui
- Zustand (상태 관리)
- React Hook Form + Zod (폼 검증)
- Recharts (차트)
- Tiptap (에디터)
- qrcode.react (QR 코드)

### 백엔드 연동 제외
- Supabase 연동은 UI만 구현, 실제 API 호출은 나중에 추가
- 더미 데이터로 UI 테스트
- Mock 함수로 API 호출 시뮬레이션

### 우선순위
1. **Phase 1-3**: 핵심 UI (인증, 레이아웃, 방/팀 관리) - 먼저 완료
2. **Phase 4**: 분석 기능 UI - 두 번째
3. **Phase 5**: 문서 및 고급 기능 - 세 번째
4. **Phase 6**: 최적화 - 마지막
